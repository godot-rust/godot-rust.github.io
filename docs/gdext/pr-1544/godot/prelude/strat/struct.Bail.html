Struct Bail 

pub struct Bail { /* private fields */ }
Expand description

Ergonomic catch-all error type for #[func] methods.

Bail is intended for unexpected errors. It always prints an error in Godot, but GDScript code is not prepared to handle it.

Its main advantage over other ErrorToGodot types is that it preserves type safety: GDScript’s static analysis sees the return type T. This is the (more common) happy path, returned from Rust as Ok(T).

Bail enables automatic conversions from other errors via ? operator. This means you can mix different error types within the same function body – each one propagates via ? and its message is forwarded to Godot. String and &str also convert, for cases where you want to fail with a direct message.

Use the func_bail! macro for early returns with an error message.

§Example

use godot::prelude::*;

#[godot_api]
impl PlayerData {
    // Load savefile and return high score.
    // GDScript receives int score. If anything goes wrong, the call fails with a Godot error.
    #[func]
    fn load_high_score(&self, save_path: GString) -> Result<i64, strat::Bail> {
        // std::io::Error and ParseIntError both convert via ? -- no map_err() needed.
        let text = std::fs::read_to_string(save_path.to_string())?;
        let score = text.trim().parse::<i64>()?;

        if score < 0 {
            func_bail!("Corrupted save file {save_path}: high score is negative.");
        }

        Ok(score)
    }
}

§Calling conventions and observable behavior

When calling Rust from GDScript, the GDScript VM picks a calling convention based on static type information available at the call site. GDExtension cannot override this.

  • Untyped object: var obj: Variant = ...
    The varcall (variant call) convention is used. In case of Err, godot-rust uses GDExtension’s r_error output parameter.
    • In Godot debug/editor builds, the GDScript VM honors this and aborts the calling GDScript function.
    • In Godot release builds, r_error is ignored, so execution continues and the failing Rust function returns the default value.
  • Typed object: var obj: MyClass = ...
    The ptrcall (pointer call) convention is used. An Err results in the caller receiving a default value on GDScript side (null in case of objects and variants).

We recommend to use Bail only for unexpected/logic errors – where you would use panics in Rust. If GDScript should handle errors, use a non-fatal strategy; see error::strat module.

§std::error::Error and trait coherence

This type intentionally does not implement std::error::Error – the reason is a Rust coherence constraint.

If Bail implemented Error, the blanket impl<E: Into<Box<dyn Error + ...>>> From<E> for Bail would conflict with the standard library’s impl<T> From<T> for T, because Bail would satisfy both E = Bail and Bail: Into<Box<dyn Error>>. Omitting the Error impl sidesteps this conflict and is the same technique used by anyhow::Error.

Because Bail is typically the last error in a chain – returned to Godot via #[func] – the lack of direct error APIs is usually not a big problem.
