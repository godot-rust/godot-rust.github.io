Struct FuncError 

pub struct FuncError { /* private fields */ }
Expand description

Ergonomic catch-all error type for #[func] methods.

FuncError is a fatal error type: when returned from a #[func] method, it causes the GDScript function call to fail and prints the error message as a Godot error.

Its main advantage over other ErrorToGodot types is that it preserves type safety: GDScript’s static analysis sees the return type T. This is the (more common) happy path, returned from Rust as Ok(T).

FuncError enables automatic conversions from other errors via ? operator. This means you can mix different error types within the same function body – each one propagates via ? and its message is forwarded to Godot. String and &str also convert, for cases where you want to fail with a direct message.

Use FuncResult<T> as an alias for Result<T, FuncError>, and the func_bail! macro for early returns with an error message.

§Example

use godot::meta::error::{FuncResult, func_bail};

#[godot_api]
impl PlayerData {
    // Load savefile and return high score.
    // GDScript receives the score directly; if anything goes wrong, the call fails with a Godot error.
    #[func]
    fn load_high_score(&self, save_path: GString) -> FuncResult<i64> {
        // std::io::Error and ParseIntError both convert via ? -- no map_err() needed.
        let text = std::fs::read_to_string(save_path.to_string())?;
        let score = text.trim().parse::<i64>()?;

        if score < 0 {
            func_bail!("Corrupted save file {save_path}: high score is negative.");
        }

        Ok(score)
    }
}

§Recoverable errors

FuncError is a fatal strategy only. If you need GDScript to inspect the error value rather than having the call fail, use global::Error, ErrorAsVariant, or a custom ErrorToGodot impl instead. See the ErrorToGodot docs for a comparison of all available strategies.

§std::error::Error and trait coherence

This type intentionally does not implement std::error::Error – the reason is a Rust coherence constraint.

f FuncError implemented Error, the blanket impl<E: Into<Box<dyn Error + ...>>> From<E> for FuncError would conflict with the standard library’s impl<T> From<T> for T, because FuncError would satisfy both E = FuncError and FuncError: Into<Box<dyn Error>>. Omitting the Error impl sidesteps this conflict and is the same technique used by anyhow::Error.

Because FuncError is typically the last error in a chain – returned to Godot via #[func] – the lack of direct error APIs is usually not a big problem.
