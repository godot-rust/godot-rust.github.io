Trait ErrorToGodot 

pub trait ErrorToGodot<T>: Sized
where
    T: ToGodot,
{
    type Mapped: GodotConvert;

    // Required methods
    fn ok_to_via(val: &T) -> <Self::Mapped as GodotConvert>::Via;
    fn err_to_via(&self) -> Result<<Self::Mapped as GodotConvert>::Via, String>;
}
Expand description

Defines how a Rust error type maps to Godot when used in Result<T, E> return types of #[func] methods.

The associated type Mapped determines what GDScript sees as the function’s return type. This type can depend on T – the ok-value type of the Result – because the trait is generic over T.

§Usage

Users implement two required methods (ok_to_via and err_to_via) and set the Mapped associated type. Use a blanket impl over T to cover all possible ok-types at once.

§Failing function call + error message

One option is to make the GDScript function call fail, meaning the GDScript function will be aborted. This happens when you use String error type. The declared type on GDScript side is T.

For a more ergonomic catch-all approach, use FuncError which lets you use ? with any impl Error type.

#[godot_api]
impl MyNode {
    #[func]
    fn load_config(&self, path: GString) -> Result<i64, String> {
        let data = std::fs::read_to_string(path.to_string())
            .map_err(|e| format!("read failed: {e}"))?;

        data.trim().parse::<i64>()
            .map_err(|e| format!("parse failed: {e}"))
    }
}

§global::Error

Another is to use the dedicated Godot enum global::Error. This only allows unit types for the Ok path.

#[godot_api]
impl MyNode {
    #[func]
    fn do_fallible_work(&self) -> Result<(), global::Error> {
        // ...
        Ok(())
    }
}

§Custom implementation: typed Array<T> with 0 or 1 elements

Since the trait is generic over T, custom implementations can require tighter bounds (such as Element) and use a typed Array<T> as the via type. This example returns a 1-element array on success and an empty array on error. GDScript sees Array[T] as the return type.

use godot::builtin::Array;
use godot::meta::error::ErrorToGodot;
use godot::meta::{Element, GodotConvert, ref_to_arg, ToGodot};

struct MyError(String);

impl<T: Element> ErrorToGodot<T> for MyError {
    // GDScript sees `Array[T]` -- a typed array.
    type Mapped = Array<T>;

    fn ok_to_via(val: &T) -> Array<T> {
        array![ref_to_arg(val)]
    }

    fn err_to_via(&self) -> Result<Array<T>, String> {
        Ok(Array::new())
    }
}

GDScript usage:

var result: Array[int] = node.try_something()
if result.is_empty():
    print("Operation failed")
else:
    var value = result[0]

Required Associated Types§

type Mapped: GodotConvert

The type to which Result<T, Self> is mapped on Godot side.

Required Methods§

fn ok_to_via(val: &T) -> <Self::Mapped as GodotConvert>::Via

Convert an Ok value to the result’s Godot representation.

For fatal mode (Mapped = T), this is typically val.to_godot_owned().

fn err_to_via(&self) -> Result<<Self::Mapped as GodotConvert>::Via, String>

Handle the error: return a Godot representation (Ok) or notify Godot that the call failed (Err).

For fatal error strategies (e.g. Mapped = T), this should return Err(message). The GDScript->Rust function call fails and the message will be printed as a Godot error.

For recoverable strategies, this should return Ok(value) matching Mapped.

Dyn Compatibility§

This trait is not dyn compatible.

In older versions of Rust, dyn compatibility was called "object safety", so this trait is not object safe.
